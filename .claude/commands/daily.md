# /daily — 일일 리포트

## 핵심 원칙
- 형식 강제 없음. 사용자가 편한 대로 아무렇게나 보내면 Claude가 알아서 파싱.
- 30초 팩트 + 2분 심층 질문 1개 = 최대 2.5분.
- 추가 질문은 1개만. 2개 이상 절대 금지.

## Step 1: 사용자 입력 (형식 자유)

어떤 형식이든 받는다:

```
"오늘 아침 쉐이크 먹고 점심 제육 저녁 닭볶 야식은 안먹음
알고리즘 풀었고 CS는 운체 봤음 임베디드는 안함
운동 상체A 했고 게임은 롤토체스 한판 함
1시반에 잠"
```
```
"쉐이크/제육/닭볶/X 알고O CSO 임베X 운동O상체A 롤토1 잠0130"
```
```
"오늘 별로였음 공부 알고리즘만 하나 풀고 나머진 다 못함 게임만 함"
```
```
"아 점 저 야 다 대충 먹음 공부 안함 운동도 안함 게임만 3판함 3시에 잠"
```

전부 유효. Claude가 문맥으로 파싱. 못 알아먹겠는 부분은 null.

## Step 2: 파싱 → Notion 기록

사용자 입력에서 추출 가능한 것만 추출:

| 필드 | 추출 방법 |
|------|----------|
| Meals | 아침/점심/저녁/야식 관련 텍스트 |
| Activities | 공부/학습 관련 언급 전부 자유텍스트로 |
| Exercise | 운동 관련 언급 (종목, 유무) |
| Gaming | 게임 언급 (종류, 판수) |
| Sleep Time | 취침 시간 |
| Wake Time | 기상 시간 (언급 시) |
| Mood | 명시적 숫자 또는 감정 표현에서 추론 ("별로" → 2, "좋았음" → 4) |
| Energy | 명시적 숫자 또는 표현에서 추론 |
| Freeform | 위 카테고리에 안 들어가는 모든 텍스트 |
| Source | "Manual" |

**Mood/Energy 추론 규칙:**
- 명시적 숫자 있으면 그대로 사용
- "별로/최악/지침" → Mood 2, Energy 2 (추론 표시)
- "좋았음/괜찮" → Mood 4
- 아예 언급 없으면 → null (추론하지 않음)
- 추론한 경우 페이지 본문에 "(추론값)" 표기

## Step 3: 중복 체크 + 기록

```
notion-search(
  query: "YYYY-MM-DD",
  data_source_url: "collection://DAILY_LOG_DS_ID",
  filters: {created_date_range: {start_date: "오늘", end_date: "오늘"}},
  page_size: 5, max_highlight_length: 0
)
```
- 없으면 → notion-create-pages
- 있으면 → notion-update-page (기존 유지, 새 데이터만 업데이트)

## Step 4: 심층 질문 1개

기록 완료 후, 질문 1개를 던진다. 사용자가 안 답해도 됨.

### 질문 선택 규칙
1. 매일 다른 카테고리 (7일간 같은 카테고리 반복 금지)
2. 최근 PULSE 인사이트와 연결된 질문 우선
3. 오늘 입력에서 발견된 신호에 반응하는 질문 우선

### 카테고리별 질문 풀

**몰입** (오늘 입력에 활동이 많을 때):
- "오늘 한 것 중 제일 몰입한 건 뭐였어?"
- "시간 가는 줄 모른 순간이 있었어?"

**회피** (오늘 무언가를 안 했을 때):
- "[안 한 것]은 왜 안 했어?"
- "오늘 미루거나 피한 게 있었어?"

**감정** (Mood가 낮거나 "별로" 표현 시):
- "오늘 하루 한 단어로 표현하면?"
- "가장 짜증났거나 힘들었던 순간은?"

**에너지** (Energy 관련 신호 시):
- "오늘 에너지가 제일 높았던 때는 언제였어?"
- "에너지가 확 떨어진 순간이 있었어?"

**사회** (3일 이상 사회 데이터 없을 때):
- "오늘 누구랑 제일 오래 있었어?"
- "오늘 혼자 있고 싶었어, 누군가랑 있고 싶었어?"

**성장** (공부 관련 활동이 있을 때):
- "오늘 새로 배운 것 하나만 말해줄래?"
- "어려웠던 건 뭐였어?"

**메타** (주 1회, 주로 일요일):
- "이번 주 나한테 한마디 한다면?"
- "내일 가장 하기 싫은 건?"

**PULSE 연동 예시:**
- "수요일 에너지 하락" 인사이트 존재 + 오늘이 수요일
  → "오늘 수요일인데, 에너지 어때? 특별히 지친 느낌 있어?"
- "임베디드 몰입" 인사이트 존재 + 오늘 임베디드 안 함
  → "임베디드 안 했는데, 이유가 있어?"

### 답변 처리
- 답변 있으면 → 해당 필드에 반영 (Approached/Avoided/Social/Freeform 등)
- "패스" 또는 무응답 → 기록 없음. 추가 질문 안 함.
- 길게 답하면 → 핵심만 추출, 전문은 페이지 본문에 기록.

## Step 5: Calendar Gap (선택)

```
gcal_list_events(
  calendarId: "primary",
  timeMin: "YYYY-MM-DDT00:00:00",
  timeMax: "YYYY-MM-DDT23:59:59",
  timeZone: "Asia/Seoul"
)
```
→ Plan-Reality Gap 계산 → update

## Step 6: 응답

```
"✅ 기록 완료. 🔥 N일 연속!
 [Gap 있으면 1줄 요약]
 [심층 질문 1개]"
```

사용자가 질문에 답하면 → 추가 기록 후 "고마워, 기록됨." 으로 끝.
답 안 하면 → 끝. 다음 날로.

## 전체 소요: 30초(팩트) + 0~2분(질문 답변) = 최대 2.5분
