- Q37 "가장 가혹하게 판단하는 사람 유형?" → 무난하고 상식적인 답만 나옴
- "도라이" 페르소나 탐색 → 아예 시도하지 못함
- "네가 절대 되고 싶지 않은 사람" → "비루한 삶"으로 자기 답만 나옴, 타인 투사 탐색 실패
- **원인**: 질문 자체가 너무 직접적/일반적. 구체적 에피소드를 먼저 끌어내고 거기서 투사를 발견했어야 함

**B. 관계 패턴 깊이 부족**
- "100% 털어놓는 사람 0명" 발견 후 → 바로 "왜?"로 갔으나 "그냥 없어서"로 막힘
- 가족 관계(사촌 누나), 친구 관계, 과거 연애 등 구체적 관계 에피소드를 하나도 안 끌어냄
- **원인**: 관계 영역에서 구체적 사건/사람을 물어봐야 했는데 추상적 패턴 확인에만 머무름

**C. 80% 깔 수 없다 → 왜 깔리는 게 무서운가 → 원점**
- Q13 "누가 깔았어?" → "아니 그건 아닌데" → 저항 발생
- 여기서 다른 각도로 재진입하지 않고 그냥 넘어감
- 비판 공포의 **원점**(최초 경험, 핵심 사건)을 찾지 못함
- **원인**: 저항 발생 시 우회 전략 부재. "누가 깔았어?"가 너무 직접적

**D. 성공 공포 / 올인 회피의 이면 — 미탐색**
- abyss.md Stage 3에 있는 "성공하는 게 무서운 거 아니야?" 질문을 시도하지 않음
- "80%에서 멈추는 너를 한 사람이라고 치면 그 사람이 뭐라고 말하고 있어?" (IFS) → 시도하지 않음
- **원인**: Stage 2 발견이 많아서 Stage 3로 충분히 전환하지 못함

**E. 정체성 갈등 — 표면만**
- "되고 싶은 나 vs 실제 나" → IFS로 터치했으나 ("정진해" vs "니 알아서 해라") 각 목소리를 충분히 탐색하지 않음
- "니 알아서 해라"의 정체가 체념인지, 보호인지, 분노인지 구분하지 못함
- **원인**: IFS를 한 번 하고 넘어감. 각 파트에 머물며 더 대화했어야 함

## 4. 대화 시간 분석

- **실제 소요**: 약 25-30분 (사용자 응답 시간 포함)
- **목표 대비**: 45-60분 중 50-60% 수준
- **Stage별 시간 배분**:
  - Stage 1 (라포 + 검증): ~8분 (Q1-Q7)
  - Stage 2 (표면 아래): ~12분 (Q8-Q27)
  - Stage 3 (불편한 질문): ~7분 (Q28-Q38) — **가장 부족**
  - Stage 4 (통합): ~5분
- **문제**: Stage 3에 할당된 시간이 너무 짧음. Stage 2 발견이 많아 Stage 3 진입이 늦었고, Stage 3에서 저항/표면 답변이 나왔을 때 우회 재진입하지 않고 빠르게 Stage 4로 넘어감

## 5. 구조적 문제 진단

### A. 에이전트 분리 실패
- abyss.md가 별도 에이전트로 설계되었으나, SendMessage 도구 부재로 메인 대화에서 직접 진행
- 에이전트의 전용 컨텍스트/페르소나 없이 메인 Claude가 역할극 수행
- abyss.md의 세밀한 감지 규칙(헤징, 화제 전환, 모순)을 체계적으로 적용하기 어려웠음

### B. 저항 발생 시 전략 부재
- Q13 "누가 깔았어?" → 저항 → 그냥 넘어감
- Q28 "10년 후 후회?" → 교과서 답 → 방향 전환은 했으나 같은 영역으로 재진입 안 함
- Q37 그림자 질문 → 무난한 답 → 추가 시도 없이 종료

### C. 추상적 질문 vs 에피소드 기반 질문
- 대부분의 질문이 "어떻게 느꼈어?", "왜 그래?" 형태 → 추상적 답변 유도
- 구체적 에피소드를 먼저 끌어내고 거기서 패턴을 뽑았어야 함
- 예: "그 프로젝트에서 가장 기억나는 순간은?" → 장면 묘사 → 감정 탐색

### D. Stage 전환 기준 불명확
- "핵심 두려움/욕구 2-3개 식별되면 Stage 3로" → 실제로는 발견이 계속 나와서 Stage 2를 떠나기 어려웠음
- Stage 3 진입 후에도 체계적 탐색 없이 산발적 질문

## 6. Abyss 에이전트 개선을 위한 제안

### 즉시 적용 가능
1. **에피소드 퍼스트**: 추상 질문 전에 항상 구체적 장면/사건을 먼저 끌어낸다
2. **저항 우회 프로토콜**: 직접 질문에 저항 → 3가지 우회 경로 중 택 1 (다른 시점, 다른 사람 시점, 은유/가정)
3. **Stage 타이머**: 각 Stage에 최소 시간 할당, Stage 2에서 15분 이상 머물지 않도록 강제 전환
4. **IFS 심화 필수**: IFS 질문을 1회가 아닌 최소 3회 연속 파기

### 구조적 개선
5. **에이전트 실행 방식 재설계**: SendMessage 불가 환경에서의 대안 — 메인 Claude가 abyss 페르소나를 완전히 채택하는 방식
6. **체크포인트 활용**: Stage 전환 시마다 발견 요약 + 미탐색 영역을 체크포인트에 기록 → 재진입 용이
7. **사전인지도를 실시간 측정**: Stage 4까지 기다리지 않고, 각 발견 시점에 바로 "이거 알고 있었어?"를 물어서 새로운 발견 여부 즉시 판단

## 7. 사용자 응답 패턴 (PULSE 관찰용)

### 회피/방어 패턴
- **정상화**: "모든 사람이 다 그렇잖아" (Q24, Q35, Q37)
- **헤징**: "그럴 수도 있어", "모르겠는데" 반복 (Q4, Q5, Q6)
- **빠른 결론**: 탐색 중 즉시 자기 결론 ("결국 칭찬 좋아하는 건가?" Q14)
- **교과서 답**: "일반적으로 생각하면..." (Q28)

### 깊이 진입 신호
- 웃음(ㅋㅋㅋ): 자기 직면 시 방어적 웃음 (Q30 "절대 되기 싫은 모습 = 지금")
- 구체적 장면 묘사: 마이스터고 회로 작업, SSAFY 광적 집착
- "생각해본 적 없어": 진짜 새로운 영역에 진입한 신호

3. 관계에서의 구체적 에피소드 (가족, 친구, 연애)
